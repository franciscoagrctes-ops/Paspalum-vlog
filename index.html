Sistema abierto

Diseño claro, datos editables.

La página queda pensada como artefacto estático de GitHub Pages: fácil de publicar, simple de modificar y lista para que Open Design use el archivo DESIGN.md como contrato visual.

  1. 1Subir datos. Convertimos CSV, GeoJSON o KML/KMZ a puntos web.
  2. 2Revisar especies. Los nombres científicos se escriben en cursiva, por ejemplo Paspalum guenoarum.
  3. 3Publicar. GitHub Pages sirve el blog sin backend ni dependencias complejas.
